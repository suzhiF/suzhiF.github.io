MySQL 索引失效情况

2021-08-09 MySQL

User 表：

1：查询的数量是大表的大部分，30％以上，索引失效

2：like 以 % 开头，索引失效

3：or 只要其中一个条件没有索引，索引失效

4：对于多列索引，不是使用的第一部分(第一个)，索引失效

5：数据类型出现隐式转化。如varchar不加单引号的话可能会自动转换为int型，索引失效

6：当全表扫描速度比索引速度快时，mysql会使用全表扫描，此时索引失效

7：索引列进行运算.需要建立函数索引
